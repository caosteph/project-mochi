# Phase 0 — Scaffolding & Message Loop (Build Steps)

**Goal of this phase:** stand up the skeleton so you can text your agent from your phone and get a reply from a *local* model — and have that conversation survive a restart. No memory, no Gmail, no proactivity yet. Just the loop, wired end-to-end and durable.

**Milestone (definition of done):**
1. You message the bot from Telegram on your phone → you get a reply generated by a local model running on your Mac.
2. A message from any chat that isn't your whitelisted `chat_id` is ignored.
3. You stop the app, restart it, and the conversation state (message history) is still there.

**Est. time:** a focused half-day, mostly install/setup. The code is small.

---

## Overview of the pieces you'll stand up

```
Your phone (Telegram)
      │  long-polling (no public URL needed)
      ▼
telegram.py  ── chat_id whitelist ──▶  graph.py (LangGraph)
                                          │
                                    ┌─────┴─────┐
                                    ▼           ▼
                              Ollama       Postgres
                          (local model)   (checkpointer = durable state)
```

Five things to install/configure: **Python 3.12**, **Ollama** (+ a model), **Postgres.app** (+ pgvector), a **Telegram bot token**, and the **project code**.

**Postgres.app** (database): download from https://postgresapp.com, drag to Applications, open it, click **Initialize** to create a server. Add its CLI tools to your PATH (Postgres.app shows the exact line; typically):

```bash
sudo mkdir -p /etc/paths.d && echo /Applications/Postgres.app/Contents/Versions/latest/bin | sudo tee /etc/paths.d/postgresapp
# open a new terminal, then:
psql --version
```

## Step 2 — Create the Telegram bot (gets you a token + your chat_id)

1. In Telegram, message **@BotFather** → `/newbot` → follow prompts → it gives you a **bot token** like `123456:ABC-DEF...`. Keep it secret.
2. Get your own **chat_id**: message your new bot anything, then visit
   `https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates` in a browser — find `"chat":{"id":<NUMBER>}`. That number is your `TELEGRAM_CHAT_ID`.

> The `chat_id` whitelist is your first security control: the bot will only respond to *you*, even though anyone could theoretically find the bot.

## Step 5 — The code (minimal, ~100 lines total)

**`app/config.py`** — one place for all settings; fails loudly if a secret is missing.
```python
from pydantic_settings import BaseSettings, SettingsConfigDict

class Settings(BaseSettings):
    model_config = SettingsConfigDict(env_file=".env")
    telegram_bot_token: str
    telegram_chat_id: int
    database_url: str = "postgresql://localhost/personal_agent"
    ollama_base_url: str = "http://localhost:11434/v1"
    local_model: str = "qwen2.5:7b"
    local_only: bool = True

settings = Settings()
```

**`app/agent/graph.py`** — a single-node graph that calls the local model, with a durable Postgres checkpointer.
```python
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.checkpoint.postgres import PostgresSaver
from langchain_openai import ChatOpenAI
from app.config import settings

llm = ChatOpenAI(
    base_url=settings.ollama_base_url,
    api_key="ollama",                 # Ollama ignores the key but the client wants one
    model=settings.local_model,
)

def agent_node(state: MessagesState):
    return {"messages": [llm.invoke(state["messages"])]}

def build_app():
    g = StateGraph(MessagesState)
    g.add_node("agent", agent_node)
    g.add_edge(START, "agent")
    g.add_edge("agent", END)
    # checkpointer persists state to Postgres → survives restarts
    checkpointer = PostgresSaver.from_conn_string(settings.database_url)
    checkpointer.setup()              # creates checkpoint tables (first run only)
    return g.compile(checkpointer=checkpointer)
```

**`app/channels/base.py`** — the seam that lets us swap Telegram for iMessage later.
```python
from abc import ABC, abstractmethod

class Channel(ABC):
    @abstractmethod
    def run(self): ...
```

**`app/channels/telegram.py`** — long-polling adapter with the whitelist.
```python
from telegram import Update
from telegram.ext import Application, MessageHandler, filters, ContextTypes
from langchain_core.messages import HumanMessage
from app.config import settings
from app.agent.graph import build_app
from app.channels.base import Channel

class TelegramChannel(Channel):
    def __init__(self):
        self.agent = build_app()

    async def _on_message(self, update: Update, ctx: ContextTypes.DEFAULT_TYPE):
        if update.effective_chat.id != settings.telegram_chat_id:
            return                                  # whitelist: ignore everyone else
        text = update.message.text
        # thread_id keys the durable conversation; one per chat for now
        cfg = {"configurable": {"thread_id": str(update.effective_chat.id)}}
        result = self.agent.invoke({"messages": [HumanMessage(text)]}, cfg)
        reply = result["messages"][-1].content
        await update.message.reply_text(reply)

    def run(self):
        app = Application.builder().token(settings.telegram_bot_token).build()
        app.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, self._on_message))
        app.run_polling()                           # long-polling: no public URL needed
```

**`app/main.py`** — entrypoint.
```python
from app.channels.telegram import TelegramChannel

if __name__ == "__main__":
    TelegramChannel().run()
```

## Step 7 — Verify the milestone (the three checks)

1. **Local reply works.** Message the bot; confirm you get a coherent response. Optional proof it's local: turn off Wi-Fi and it still replies.
2. **Whitelist works.** Have a friend message the bot (or use a second Telegram account) → the bot stays silent. Only your `chat_id` gets a response.
3. **Durability works.** Send a couple of messages, `Ctrl-C` to stop the app, restart with `uv run python -m app.main`, then send "what did I just say?" — because state lives in Postgres, the prior turns are still in context.

## Common gotchas

- **`connection refused` to Ollama** → `ollama serve` isn't running, or the model isn't pulled. Check `ollama list`.
- **Postgres connection errors** → Postgres.app server isn't started, or `personal_agent` DB doesn't exist (`createdb personal_agent`).
- **`vector` extension missing** → run the `CREATE EXTENSION` from Step 3 (harmless in Phase 0, needed Phase 1).
- **Bot doesn't respond at all** → wrong token, or `TELEGRAM_CHAT_ID` doesn't match your account (re-check `getUpdates`).
- **Reply is very slow** → normal for a cold model on M2; it speeds up after warm-up. If persistently painful, try a smaller quant.

---

## What Phase 0 deliberately does NOT do (comes next)

- **No memory** — it only remembers within a conversation thread via the checkpointer, not durable facts about your life. That's **Phase 1** (Postgres schema + local embeddings + hybrid recall + `remember_fact`/`recall` tools).
- **No Gmail/Calendar** — **Phase 2**.
- **No proactivity / reminders** — **Phase 3**.
- **No model router** — everything is local now (`LOCAL_ONLY=true`); the deterministic local-vs-hosted router arrives when non-sensitive building starts (Phase 4).
